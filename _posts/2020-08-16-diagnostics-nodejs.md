Prior to v12.8.3, in order to get a substantial amount of information about what Node was doing, you would need to use a number of modules like `os`, `process`, and custom npm libraries for heap and v8 information. Luckily, now we have a `report` built-in module to generate full fledge diagnostic reports with loads of information such as:

* Versioning (glib, arch, platform, node, module versions)
* OS Information
* CPU Details (speed, model, current timings)
* Network Interfaces
* Host
* Stack trace
* Native Stack
* Heap Usage Statistics
* Resource Usage (file system, disk, cpu, page faults)
* uvthread Usage
* libuv statistics
* Environment variables
* user limits (configured limits on the system)
* shared objects / libraries

I can tell you one of the most useful items on this list is the user limits. I've run into problems so many times where I forgot to configure these extra limits in the deployment that it's great to just have this information right in the report when an error crops up or when we need a report on demand like this.

## Use

Usage is very straightforward, it's right on the `process` object.

```
process.report.writeReport()
```

This will generate a report to a JSON file, you can pass along the file name in the `writeReport` argument. Alternatively you can call `getReport` as an object.

```
const report = process.report.getReport()
console.log(JSON.stringify(report, null, 2));
```

Optionally, you can pass along a new `Error` to the `getReport` function if you want to add some context to the stack trace.

## Runtime Generation

You can also add some runtime configuration arguments to automatically generate reports according to various signals and conditions such as:

```
// Trigger report only on uncaught exceptions.
process.report.reportOnFatalError = false;
process.report.reportOnSignal = false;
process.report.reportOnUncaughtException = true;

// Trigger report for both internal errors as well as external signal.
process.report.reportOnFatalError = true;
process.report.reportOnSignal = true;
process.report.reportOnUncaughtException = false;

// Change the default signal to 'SIGQUIT' and enable it.
process.report.reportOnFatalError = false;
process.report.reportOnUncaughtException = false;
process.report.reportOnSignal = true;
process.report.signal = 'SIGQUIT';
```

These can be specified through the node options as well.

```
NODE_OPTIONS="--report-uncaught-exception \
  --report-on-fatalerror --report-on-signal \
  --report-signal=SIGUSR2  --report-filename=./report.json \
  --report-directory=/home/nodeuser"
```
